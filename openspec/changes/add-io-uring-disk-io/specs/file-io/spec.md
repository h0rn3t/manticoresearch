# file-io

## ADDED Requirements

### Requirement: Async-чтение через io_uring в корутинном контексте

Система MUST читать doclists/hitlists в режиме `FileAccess_e::FILE` внутри
корутины через io_uring вместо блокирующего `sphPread`, когда io_uring
включён и доступен. Корутина MUST отправлять чтение в кольцо и приостанавливаться
(`YieldWith`), освобождая worker-поток; завершение (CQE) ДОЛЖНО пробуждать ту же фибру
через `Waker_c`.

#### Scenario: Чтение doclist под корутиной с включённым io_uring

- **GIVEN** запущенный `searchd` с `io_uring = 1` на ядре с поддержкой io_uring
- **AND** индекс с `access_doclists = file`
- **WHEN** выполняется полнотекстовый запрос, читающий doclist с диска
- **THEN** чтение отправляется в io_uring (submit), а worker-поток не блокируется на ядре
- **AND** результат запроса идентичен результату при чтении через `sphPread`

#### Scenario: Завершение чтения пробуждает корректную фибру

- **GIVEN** несколько одновременных запросов с inflight io_uring-чтениями
- **WHEN** io-поток-reaper получает CQE
- **THEN** он пробуждает именно ту фибру, что отправила соответствующий SQE (по `user_data`)
- **AND** фибра продолжает с корректным числом прочитанных байт или ошибкой из `cqe.res`

### Requirement: Fallback на pread

Система MUST (ДОЛЖНА) сохранять путь чтения через `sphPread` и использовать его, когда io_uring
выключен (`io_uring = 0`), недоступен на рантайме, или чтение происходит вне корутинного
контекста. Недоступность io_uring MUST NOT (НЕ ДОЛЖНА) приводить к отказу запуска.

#### Scenario: io_uring заблокирован seccomp

- **GIVEN** контейнер, где `io_uring_setup` запрещён seccomp-профилем
- **WHEN** `searchd` стартует с `io_uring = 1`
- **THEN** детект фиксирует недоступность, логирует варнинг и откатывается на `sphPread`
- **AND** `searchd` стартует и обслуживает запросы

#### Scenario: Чтение вне корутины

- **GIVEN** код, читающий файл вне корутинного контекста (например, при загрузке/сервисных задачах)
- **WHEN** выполняется чтение
- **THEN** используется `sphPread` (синхронный путь), независимо от настройки io_uring

### Requirement: Конфигурация io_uring

`searchd` MUST (ДОЛЖЕН) принимать опцию `io_uring` (вкл/выкл, с авто-детектом доступности).
Опция управляет async-путём дискового чтения и используется для сравнительных бенчей на
одном бинаре.

#### Scenario: Переключение пути чтения через конфиг

- **GIVEN** один и тот же бинарь `searchd`
- **WHEN** оператор задаёт `io_uring = 0` и перезапускает
- **THEN** всё дисковое чтение `FILE`-режима идёт через `sphPread` (бэйзлайн)
- **AND** при `io_uring = 1` — через io_uring (если доступен)
